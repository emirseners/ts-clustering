simgon.distance_pattern_dtw.distance_pattern_dtw(data, significanceLevel=0.01, wSlopeError=1, wCurvatureError=1)[source]ÔÉÅ

Calculate pairwise pattern distances between all data sequences using Dynamic Time Warping.

The distance measures the proximity of data series in terms of their qualitative pattern features. In other words, it quantifies the proximity between two different dynamic behaviour modes by allowing for time warping.

Parameters:

  • data (np.ndarray) ‚Äì 2D array of shape (n_samples, n_features) containing sequences to compare.

  • significanceLevel (float, optional) ‚Äì The threshold value to be used in filtering out fluctuations in the slope and the curvature (default=0.01).

  • wSlopeError (float, optional) ‚Äì Weight of the error between the slope dimensions (first dimension) of the two feature vectors (default=1).

  • wCurvatureError (float, optional) ‚Äì Weight of the error between the curvature dimensions (second dimension) of the two feature vectors (default=1).

Return type:

Tuple[ndarray, List[Tuple[Dict[str, str], ndarray]]]

Returns:

  • dRow (np.ndarray) ‚Äì Condensed distance matrix as 1D array of length n_samples * (n_samples - 1) / 2.

  • data_w_desc (List[Tuple[Dict[str, str], np.ndarray]]) ‚Äì List of (metadata_dict, sequence) tuples for tracking original data and feature vectors.

Internal Dependencies

This module internally uses functions from simgon.behavior_splitter for feature extraction:

  • simgon.behavior_splitter.construct_features() - Constructs feature vectors for time series data based on behavioral characteristics

  • simgon.behavior_splitter.split_behavior() - Splits a time series into sections of different atomic behavior modes

These functions are used to extract behavioral features (slope and curvature signs) from time series data before computing pattern DTW distances. While end users don‚Äôt directly call these functions, they can access the source code to understand how the behavioral features are extracted and processed.
